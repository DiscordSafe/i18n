Guía detallada sobre la configuración paso a paso de todas las funciones y comandos del bot D-Safe (Discord Safe).

:::note Información
Si algo falla por favor revisa si el bot tiene permisos, si el problema persiste pregunta en [nuestro servidor de soporte](https://discordsafe.com/discord). 

Recuerde que para los comandos no debe poner `<>`, `[]` o `/`.
* El contenido dentro de `[]` indica que es obligatorio.
* El contenido dentro de `<>` indica que es opcional.
* Las `/` dividen las opciones.

Para la mayoría de comandos que se mencionan debajo el usuario necesita permisos de **Administrador**.
:::

### Establecer el modo
Establezca el modo de verificación que más le convenga usando `d.verification setmode <Modo>`.
Los modos de verificación son los siguientes:

| 🔧 Modo | 📣 Descripción | 📑 Requiere | 🔐 Nivel protección |
| :--- | :--- | :--- | :--- |
| **Approval** | El usuario tiene que ser aprobado por un Administrador con `d.approve @Usuario` | Canal, Rol No Verificado, Rol Usuario. | **4** |
| **Captcha** | El usuario tiene que completar correctamente un captcha. | Canal, Rol No Verificado, Rol Usuario. | **3** |
| **Reaction** | El usuario tiene que reaccionar a un mensaje para acceder al servidor. | Canal, Rol No Verificado, Rol Usuario, Mensaje. | **2** |
| **Verification** | El usuario tiene que usar el comando `d.verifyme` para entrar. | Canal, Rol No Verificado, Rol Usuario. | **1** |

### Establecer el canal
Debe establecer el canal de verificación donde se mandará el captcha, o donde esté el mensaje al que tengan que reaccionar o donde tienen que poner los comandos.

Establezca el canal con `d.verification setchannel #Canal`.

### Establecer el rol de No verificado
:::caution Aviso
Usted tendrá que configurar este rol en su servidor para que solo vea los canales que usted quiera y que cuando se verifique tenga acceso completo al servidor.
:::

Este será el rol que se les dará a los nuevos usuarios al entrar al servidor antes de verificarse.

Establezca el rol de No verificados con:
`d.verification setguest @Rol`.

:::caution Aviso

### Establecer el mínimo de edad
Establezca el **mínimo de días que deben tener las cuentas** cuando se unen a su servidor, así evita que entren multicuentas o auto-bots de spam.

* Configure el mínimo con: `d.verification setage <1-30 Días>`.
* Puede desactivarlo con `d.verification setage disable`.

### Establecer máximo de intentos
:::caution Aviso
**Este es un ajuste opcional** del modo **captcha**.

Configure el máximo de intentos con ``d.verification setattempts [Número]``.

Puede volver a la cantidad por defecto con ``d.verification setattempts default``.

### Acciones en masa
Puedes aceptar, denegar y borrar sugerencias en masa si lo deseas, estos comandos no admiten comentarios.

* Aceptar: `d.suggestions massapprove IDSugerencia IDSugerencia...`.
* Denegar: `d.suggestions massdeny IDSugerencia IDSugerencia...`.
* Borrar: `d.suggestions massdelete IDSugerencia IDSugerencia...`.

### Paleta customizada
Customize la paleta de colores con `d.suggestions setpalette p1 #ColorHexadecimal`.

Puede obtener el código de color hexadecimal de la siguiente [página](https://htmlcolorcodes.com/).

Debe especificar el color de cada uno de los siguientes niveles, la `p` indica positivo, `n` indica negativo, del 1-3 cuanto más alto más votos hay.
* p1, Establece el color positivo para pocos votos.
* p2, Establece el color positivo para la mitad de votos.
* p3, Establece el color positivo para muchos votos.
* n1, Establece el color negativo para pocos votos.
* n2, Establece el color negativo para la mitad de votos.
* n3, Establece el color negativo para muchos votos.

## 🔧Whitelist
Si no quieres que un usuario malicioso sea baneado al usar `d.forceban`.

* Ignore a un usuario del **forceban** usando `d.whitelist add UsuarioID`.
* Designore a un usuario del **forceban** usando `d.whitelist remove UsuarioID`.
* Vea la lista de usuarios ignorados con `d.whitelist list`.
* Borre todos los usuarios ignorados con `d.whitelist reset`.
